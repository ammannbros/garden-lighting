Hardware - RelaisArray

The board was designed for low power consumption. The overall functionality is very simple. It offers a I2C interface to be addressed by any standard purpose Microcontroller. The boards heard is the well known MCP23017 IO-Expander(input-output) from Microchip. http://ww1.microchip.com/downloads/en/DeviceDoc/21952b.pdf

So the output registers of the chip can be written via I2C. The output registers are connected to a standard OR-Gate to allow the user to control the relys manually through a simple switch. The two inputs are the Output of the MCP23017 registers and the debounced switch. The output of the OR-Gate controles a TTL-Logic-Level MOSFET which switches the indudctors of the rails. (An EAGLE schematic is available in repository for more details)

Issues

A major Issue was, that the MCP2017 resetted after switching the relias if consumers were connected. The relais induced noise to the 12V power line. This noise caused a voltage drop on the reset line of the MCP2017. So after switching the configuration of the Chip got lost. A capacitor from the reset-pin to ground solved the problem.

Possible Improvements

If the load which is connected to the relais is known, a Snubber could be added to improve the durabiltiy of the relays. The functionality for motion detectos is already included. A potetial free switch an cause an interrupt on the MCP23017 to allow the software to react on.

Characteristics

  • Controlable via I2C
  • I2C Pullups not included (to provide the opporunity to use a Raspberry PI)
  • 12V Power supply needed
  • Designed for 230V/16A Switching load
  • 8 independet Channels

Hardware - Overall System

The Overall System include a RaspberryPI2 which runs a website to control the hardware. So a cheap 12V -> 5V stepdown converter for the raspberry power supply was ordered from china (ebay). A 12V powersuppy is needed for the RelaisArrays. All compontens are included in a IP55-Metal case so provide an earth wire protection. Some fuses were also added...
